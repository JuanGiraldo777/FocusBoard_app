# Punto 1: Investigación - Metodologías de desarrollo

## Diferencias entre Scrum y Kanban

Scrum y Kanban tienen diferencias significativas, aunque también pueden complementarse.

- **Scrum** está orientado al trabajo en equipo con roles definidos y responsabilidades repartidas según cada rol.
- **Kanban** se enfoca en la gestión de tareas mediante un flujo continuo y flexible, con menor necesidad de planificación rígida. Puede aplicarse tanto en equipos como en trabajo individual.

## Situaciones en las que se podría usar cada metodología

### Agile (app nueva tipo startup)

1. No se tienen todos los requisitos claros desde el inicio.
2. El cliente cambia de idea con frecuencia.
3. El proyecto es nuevo o innovador.
4. Se necesita iterar rápido.

### Scrum (desarrollo de apps, proyectos tipo empresa, startups organizadas)

1. Hay un equipo de trabajo.
2. El trabajo se puede dividir en entregas pequeñas.
3. Hay necesidad de planificación clara.
4. Existen fechas de entrega (deadlines).

### Kanban

1. El trabajo es continuo (no tiene inicio o fin claramente definido).
2. Llegan tareas constantemente.
3. Se necesita organización visual rápida.
4. Funciona bien para trabajo individual o equipos pequeños.
